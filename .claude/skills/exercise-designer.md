# exercise-designer Skill

## Purpose
Design hands-on coding challenges and practical exercises for skill development.

## Key Patterns

### Exercise Structure
```markdown
## Challenge: {Title}

**Difficulty**: {Beginner|Intermediate|Advanced}
**Time**: {Estimated minutes}
**Prerequisites**: {Required knowledge}

### Objective
{Clear goal statement}

### Scenario
{Real-world context or problem}

### Requirements
1. {Requirement 1}
2. {Requirement 2}
3. {Requirement 3}

### Starter Code
```python
# Provided skeleton
def your_function():
    pass  # TODO: Implement
```

### Hints
<details>
<summary>Hint 1</summary>
{Guidance without full solution}
</details>

<details>
<summary>Hint 2</summary>
{Additional guidance}
</details>

### Solution
<details>
<summary>Click to reveal solution</summary>

```python
# Complete solution with comments
def your_function():
    # Implementation details
    pass
```

**Explanation**: {Why this solution works}
</details>

### Test Cases
```python
# Verify your solution
assert your_function(input1) == expected_output1
assert your_function(input2) == expected_output2
```
```

### Progressive Difficulty
- **Week 1-3**: Fill-in-the-blank, modify existing code
- **Week 4-7**: Build from scratch with guidance
- **Week 8-13**: Open-ended design challenges

## Usage Context
- End of each chapter
- Hands-on practice
- Skill validation
- Portfolio projects
